# **产品需求文档 (PRD)：OmniSight**

## **资产全景透视与智能决策导航系统**

**版本**: V2.0 (通用版) | **最后更新**: 2026-06-01 | **撰写角色**: 资深 CPO

## **1\. 产品定位与商业价值 (Product Positioning)**

SciCompass 是一款企业级**通用型 AIGC 知识资产全景透视与策略导航系统**。系统通过解耦的分布式 n8n 工作流，支持用户批量导入任何行业的非结构化文档（如合同、研报、病历、财报等），同时定时自动抓取行业外部公共情报。利用 **GraphRAG（向量+图谱混合检索）** 消除大模型幻觉，以可视化“认知拓扑画布”与“资产词云图”直观呈现企业内部资产的“强支撑热区”与“信息/业务空白冷区（Gap Analysis）”，最终实现跨行业的智能风险预警、资产盘点与战略行动指导。

## **2\. 系统核心拓扑与解耦架构 (System Architecture)**

为了保证系统处理大批量文件时的稳定性和高可用性，后端架构必须实现“双轨三流”解耦：

1. **工作流 A（事件驱动 \- 实时内资清洗流）**：用户在前端批量上传文件时触发，主打增量去重与极速分流存储。  
2. **工作流 B（时间驱动 \- 定时外情抓取流）**：由 Cron 节点在深夜定时触发，独立运行，负责抓取全网行业前沿公开数据，与内部资产做矩阵比对。  
3. **工作流 C（查询驱动 \- GraphRAG 决策流）**：用户在输入框提问或点击画布节点时触发，主打多路混合检索与结构化报告生成。

## **3\. 核心功能模块详细需求规范 (Functional Requirements)**

### **3.1 模块一：\[升级\] 通用多模态增量清洗流水线（工作流 A）**

* **业务场景**：用户批量拖拽上传任意格式文件（PDF、Word、Excel、Markdown、图片）。  
* **实现逻辑**：  
  1. **多格式适配器**：n8n 接入通用的多模态解析节点（如 Unstructured 或 Apache Tika），统一将内容转化为标准的 Markdown 结构化文本。  
  2. **MD5 增量防线**：文件解析前计算 MD5 校验码，在 PostgreSQL Global\_Asset\_Registry 中比对，已存在的文件**立刻熔断，终止后续 AI 节点**，返回“无需重复解析”提示。  
  3. **双库分流存储**：  
     * **语义层**：文本经由 RecursiveCharacterTextSplitter（1000字符块，200重叠度）切片，通过 Embedding 转化为向量写入向量表。  
     * **关系层 (通用特征提取)**：调用 LLM 开启 JSON Mode，通用 Prompt 约束为：“你是一个通用语言学分析师，请提取该文本的核心主题实体（Entity\_A）、关联属性/关联对象（Entity\_B）以及它们之间的逻辑关系（Relationship）”。通过强制 Schema 校验后写入三元组表。

### **3.2 模块二：\[新增\] 定时行业情报自动抓取流水线（工作流 B）**

* **业务场景**：系统在后台静默运行，无需人工干预，定时补充行业大盘前沿数据。  
* **实现逻辑**：  
  1. **Cron 节点触发**：在 n8n 中配置定时触发器，设定为每日凌晨 2:00（避开用户白天使用高峰）。  
  2. **动态关键词下发**：系统自动提取当前用户数据库中高频的“前沿技术词/核心业务词”作为搜索种子。  
  3. **多源数据抓取**：调用网络爬虫或公开 API（如 arXiv、专利接口、行业新闻 RSS、财报公开渠道）异步抓取最新相关文本。  
  4. **影子库标记存储**：抓取到的外部数据采用与 工作流 A 相同的切片和特征提取机制，但存入数据库时，其 source\_type 字段必须标记为 "EXTERNAL"，与内部资产进行物理隔离。

### **3.3 模块三：通用资产认知拓扑画布与词云 (Visualization Canvas)**

* **可视化渲染**：前端基于 WebGL/ECharts 渲染力引导关系拓扑图（Force Graph）与词云图。  
* **拓扑染色与 Gap 分析机制**：  
  * **资产热区（深蓝）**：当某一技术/业务节点在内部资产（source\_type: "INTERNAL"）中的连线和三元组密度超过阈值时，该节点在画布/词云中体积变大，颜色染为**深蓝色**，代表公司在该领域有极强的知识资产支撑。  
  * **战略盲区（红色警告）**：当系统从外部抓取到某些高频技术/业务词（source\_type: "EXTERNAL"），但在内部资产中匹配度为 0 时，该节点在画布上被染成**醒目的红色**，代表企业当前在此领域存在“认知空白”或“业务盲区”。  
* **画布微交互**：用户直接点击红色的“空白节点”，系统立刻捕捉该节点的标签名称，无缝唤醒右侧侧边栏工作台。

### **3.4 模块四：GraphRAG 智能策略导航引擎（工作流 C）**

* **混合检索 (Hybrid Search)**：当用户针对某个红色空白节点发起“行动方案咨询”时，n8n 触发混合检索：  
  * **路徑 1**：向向量表发起 Top-K 语义检索，捞出外部前沿关于该空白领域的具体做法和趋势。  
  * **路徑 2**：向三元组关系表发起 SQL 查询，盘点公司内部有哪些“邻近资产/类似技术”可以作为底层支撑。  
* **生成式策略指导书**：  
  n8n 拼接上述两路上下文，投喂给高规格 LLM，强制要求输出结构化的《行业空白领域抢滩行动建议书》：  
  1. **市场/前沿大势研判**（外部目前发展到什么阶段了）  
  2. **内部可调配资产盘点**（虽然本领域是空白，但我们有哪些临近资产可以快速迁移迁移）  
  3. **差异化切入策略建议**（给出具体的立项、破局或防御性行动方案）  
  4. **破局可行性评分（0-100）**。

## **4\. 后端 n8n 独立工作流技术编排规范 (LLMOps Spec)**

研发需在 n8n 后端控制台中独立开发并部署以下三条工作流线，禁止互相嵌套调用：

### **4.1 工作流 A（实时入库线）：Webhook\_Internal\_Ingest**

\[前端批量上传/API输入\] $\\rightarrow$ \[PostgreSQL MD5 查重\] $\\rightarrow$ 满足条件 $\\rightarrow$ \[分流节点\]

* *分支 1*: \[Text Splitter\] $\\rightarrow$ \[Embedding Node\] $\\rightarrow$ \[写入 PG 向量表\]  
* *分支 2*: \[LLM Chat Node (JSON Mode)\] $\\rightarrow$ \[Schema 强校验\] $\\rightarrow$ \[写入 PG 三元组关系表\]

### **4.2 工作流 B（定时抓取线）：Cron\_External\_Spy**

\[Cron 触发 (0 2 \* \* \*)\] $\\rightarrow$ \[SQL 读取高频种子词\] $\\rightarrow$ \[HTTP Request (多源爬虫/API)\] $\\rightarrow$ \[数据清洗\] $\\rightarrow$ \[大模型特征提取\] $\\rightarrow$ \[写入 PG 三元组/向量表 (标记为 EXTERNAL)\]

### **4.3 工作流 C（智能推理线）：Webhook\_GraphRAG\_Query**

\[接收前端节点点击请求\] $\\rightarrow$ \[并行节点\]

* *并行 1*: \[Vector Store Retriever (检索外部情报)\]  
* *并行 2*: \[Execute SQL (盘点内部资产及邻近关联)\]  
  $\\rightarrow$ \[Code Node (合并上下文并拼装超级 Prompt)\] $\\rightarrow$ \[LLM Advanced Chain\] $\\rightarrow$ \[JSON Schema 拦截净化\] $\\rightarrow$ \[返回前端渲染 Markdown 报告\]

## **5\. 极致非功能性要求与防线 (NFR)**

1. **算力熔断墙 (Cost Guard)**：由于引入了凌晨自动抓取工作流（工作流 B），为了防止外部数据量过大导致 LLM 提取节点疯狂扣费，必须在工作流 B 中增设 **“单次任务 Token 阈值熔断器”**：单次凌晨任务大模型调用费用达到 20 元人民币时，工作流自动静默熔断，并向你发送警报邮件。  
2. **数据安全与物理隔离 (Data Privacy)**：SaaS 通用化后，在 PostgreSQL 中必须采用 **Row-Level Security (行级安全锁)**。基于 tenant\_id (租户ID) 或 user\_id 对数据进行严密的物理隔离，确保 A 企业上传的行业报告绝不会被 B 企业的词云或拓扑图检索到。

# **UI/UX 详细设计文档：OmniSight**

**版本**: V2.0 | **核心哲学**: 去AI化、生产力至上、克制与严谨

## **1\. 全局视觉规范 (Visual Identity & System)**

### **1.1 色彩系统 (Color Palette)**

摒弃紫色/粉色等二次元魔幻渐变，采用**硬核、学术、工业级**的冷色调：

1. **基础背景色**：纯白 (\#FFFFFF) / 极其克制的浅灰 (\#F8F9FA)。  
2. **文字与边框**：深炭黑 (\#1A1A1A) / 工业灰 (\#E5E5E5)。保证极高的阅读对比度。  
3. **状态染色（核心商务逻辑）**：  
   * 内部资产热区 (INTERNAL)：**普鲁士蓝 (\#0F4C81)**。代表沉稳、底蕴、强支撑。  
   * 外部前沿冷区 (EXTERNAL)：**暗哑红 (\#C93B2B)**。代表警告、盲区、抢滩机会。  
   * 中立关联/连线：浅钢灰 (\#D1D5DB)。

### **1.2 字体与排版 (Typography)**

* 全局使用苹果原生系统字体 San Francisco (SF Pro) 或 Inter。  
* 数据与代码部分自适应等宽字体 SF Mono。  
* **严禁使用任何发光阴影 (Glow Effect) 和悬浮粒子**，全部采用清晰的 1px 实线描边。

## **2\. 全局三栏式主布局架构 (The Grid Layout)**

整个系统采用高效的**固定三栏式**布局，信息一览无余，严禁使用层层叠加的弹窗（Modal）打断用户工作流

## **3\. 核心界面交互与视觉详述**

### **3.1 左栏：多模态资产控制舱 (Data Ingest & Registry)**

* **视觉表现**：经典的研发级列表。上方为一个支持拖拽的上传区。  
* **UI 元素**：  
  2. **拖拽上传区**：虚线描边方框。文案：“拖拽或点击上传内部资产 (支持 PDF, Word, JPG, MD)”。  
  3. **资产流水列表**：表格展示（名称、大小、状态、MD5校验码）。  
* **UX 微交互（去 AI 味的关键）**：  
  1. 用户拖入文件 $\\rightarrow$ 列表立刻新增一行，状态显示：\[MD5 校验中...\]。  
  2. 若命中所预设的增量去重防线，状态瞬间变为灰色：● 已存在 (跳过解析)，而非弹出一个巨大的“去重成功！”提示框。静默、克制。  
  3. 新文件则平滑展示 \[向量切片中 45%\] $\\rightarrow$ \[完成\]。

### **3.2 中栏：ECharts 认知拓扑画布 (The Cognitive Canvas)**

这是整个产品的核心视觉。我们把复杂的 GraphRAG 图谱和词云，融合成一个严谨的“技术/业务大盘”。

* **视觉表现**：基于 ECharts 的力引导网络拓扑图（Force Graph）。背景为干净的浅灰网格。  
* **拓扑染色规则 (Gap Analysis UX)**：  
  * 每个圆圈代表一个技术/业务实体。圆圈的大小由其在图谱中的连接密度（三元组数量）决定。  
  * **深蓝节点**：公司有大量专利、论文、数据支撑的方向。  
  * **暗红节点**：外部爬虫高频抓取，但内部资产连接数为 0 的**战略盲区**。  
* **视图切换键 (Toggle)**：画布右上角提供一组扁平的单选按钮：\[ 拓扑网格 \] | \[ 资产词云 \]。点击词云时，平滑淡出拓扑图，切换为由深蓝与暗红交织的文本词云。  
* **UX 操作流**：  
  * 用户将鼠标悬停在圆圈上 $\\rightarrow$ 浮出轻量级系统 Tooltip，展示：\[实体名称：量子计算\] | \[内部资产：0件\] | \[外部前沿：14篇\]。  
  * **点击红色节点** $\\rightarrow$ 节点产生一个轻微的缩放波纹动效（实线扩大 2px 后收回），**右栏面板平滑平移滑出**。

### **3.3 右栏：GraphRAG 智能策略导航面板 (Strategy Action Panel)**

* **视觉表现**：类似 Notion 的干净排版。顶部为面包屑导航，正文为标准、美观的 Markdown 渲染排版。  
* **状态 A：静态 (Idle)**  
  * 显示一句克制的灰色提示语：“在左侧画布点击任意‘红色空白节点’，获取 AI 抢滩立项策略指导。”  
* **状态 B：生成态 (Generating)**  
  5. 不使用带有科技感的彩色进度条。采用经典的**打字机动效 (Typing Effect)**，伴随严谨的文字一行行吐出。  
  6. 骨架屏采用平滑的灰色淡入淡出（Pulse Animation）。  
* **状态 C：终态 (Final Report)**  
  * 排版包含清晰的段落（H1, H2, H3），数据指标（如“破局可行性评分：78/100”）使用等宽大字号加粗显示。  
  * 底部常驻两个工具型按钮：\[ 复制 Markdown \]（次要按钮，线框）与 \[ 导出决策报告 (Word) \]（主要按钮，普鲁士蓝底白字）。

## **4\. 极端边缘状态与异常体验 (Edge Cases UX)**

* **海量文件撑爆画布 (Performance Edge)**：  
  * *问题*：当租户上传了超过 10,000 个文件，产生几万个实体节点时，2D 画布会卡死。  
  * *UX 解决方案*：画布顶部增加一个**阈值滑动条 (Density Filter)**：“仅显示关联度前 50/100/500 的核心节点”。超出部分默认聚合为“其他微关联”，保持画布的绝对清爽和可读性。  
* **外部抓取空转 (Cron Missing)**：  
  * *问题*：深夜凌晨定时爬虫（工作流 B）因为外部目标网站宕机或反爬，抓取失败。  
  * *UX 解决方案*：系统不在中栏画布上报错。而是在左下角的系统状态栏，静默将指示灯变为黄色：● 外部情报更新延迟 (3小时前尝试失败)。用户点击可查看错误日志，把掌控权完完全全还给科技项目管理者。
